# Publishing Checklist

Use this before moving a draft from notes into a public article.

## Article Quality

- [ ] The opening explains the problem in one paragraph.
- [ ] The article links to a real project repo when relevant.
- [ ] Commands or code snippets were tested.
- [ ] Claims are separated from plans or hypotheses.
- [ ] Limitations are explicit.
- [ ] The ending gives a concrete next step.

## Strong First Articles

1. Building an AI/MLOps portfolio with honest metric status.
2. Turning a computer-vision coursework archive into focused project repos.
3. What a reproducible ML project needs beyond a notebook.
